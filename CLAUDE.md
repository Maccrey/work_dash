# CLAUDE.md

이 파일은 Claude Code (claude.ai/code)가 이 저장소의 코드로 작업할 때 필요한 가이드라인을 제공합니다.

## 프로젝트 개요

바닐라 HTML, CSS, JavaScript로 구축된 한국어 업무 대시보드 웹 애플리케이션입니다. 날씨 알림, 작업 관리, 시간 추적, 음성 메모, 출퇴근 기록 등 다양한 기능을 제공하는 생산성 도구로 설계되었습니다.

## 개발 환경

- **빌드 프로세스 없음**: 빌드 시스템, 패키지 매니저, 종속성이 없는 간단한 정적 웹 애플리케이션
- **로컬 개발**: 브라우저에서 `index.html`을 직접 열어서 실행
- **테스트 프레임워크 없음**: 현재 자동화된 테스트가 없음
- **린팅 없음**: 린팅 설정이 존재하지 않음

## 아키텍처 및 코드 구조

### 핵심 애플리케이션 패턴
애플리케이션은 단일 JavaScript 파일 내에서 모듈 패턴을 따릅니다:
- 모든 기능은 `DOMContentLoaded` 이벤트 리스너 내에 포함됨
- 상태는 최상위 레벨의 전역 변수를 통해 관리됨
- 함수는 기능 영역별로 구성됨 (날씨, 할 일, 뽀모도로 등)
- localStorage를 사용하여 데이터 지속성 구현

### 주요 상태 변수
- `isWeatherSystemActive`: 메인 날씨 알림 시스템 제어
- `todos`, `notes`, `recordings`, `ttsSchedules`: 사용자 데이터를 저장하는 배열
- `attendanceRecords`: 날짜 키 형식(YYYY-MM-DD)으로 출근 데이터를 저장하는 객체
- `cardVisibility`: 어떤 대시보드 카드가 표시되는지 제어하는 객체
- 뽀모도로 타이머 상태 변수들 (`pomoInterval`, `pomoTimeLeft` 등)

### 데이터 지속성 패턴
모든 데이터는 다음 유틸리티 함수를 사용하여 localStorage에 저장됩니다:
```javascript
const saveData = (key, data) => localStorage.setItem(key, JSON.stringify(data));
const loadData = (key) => JSON.parse(localStorage.getItem(key));
```

### 카드 기반 UI 아키텍처
대시보드는 각 기능이 `.dashboard-card`에 포함된 카드 기반 레이아웃을 사용합니다:
- 카드는 드래그 가능하고 순서 변경 가능
- 설정을 통해 카드 표시/숨김 토글 가능
- 각 카드는 자체 DOM 조작 함수를 유지
- 카드는 고정된 크기를 가짐 (너비 380px, 높이 600px)

### API 통합
애플리케이션은 한국 정부 API와 통합됩니다:
- **날씨 API**: 기상청 API를 사용하여 날씨 데이터 가져옴
- **공휴일 API**: 출퇴근 추적을 위한 공휴일 정보 가져옴
- 한국 날씨 API 요구사항에 맞는 격자 좌표 변환 구현

### 이벤트 기반 아키텍처
- 사용자 상호작용을 위한 광범위한 이벤트 리스너 사용
- 분 단위 체크를 위한 setInterval을 사용한 시간 기반 알림
- 설정 및 구성을 위한 모달 시스템

## 주요 기능 및 컴포넌트

1. **날씨 알리미**: TTS 알림이 포함된 실시간 날씨 모니터링
2. **작업 관리**: 시간 기반 알림과 우선순위 레벨이 있는 할 일 목록
3. **뽀모도로 타이머**: 시각적, 청각적 피드백이 있는 작업/휴식 주기 추적
4. **음성 메모**: 태그 지정과 재생이 가능한 오디오 녹음
5. **출퇴근 추적**: 월간 캘린더 보기가 있는 체크인/아웃 시스템
6. **TTS 스케줄러**: 지정된 시간에 맞춤 텍스트 음성 변환 알림
7. **메모장**: 간단한 텍스트 노트 관리
8. **계산기**: 기본 계산기 기능

애플리케이션은 다음 localStorage 키를 사용합니다:

- `attendanceRecords`, `previousHourlyForecast`
- `ttsEnabled`, `todoTtsEnabled`, `pomoSettings`
- `cardVisibility`, `cardOrder`

이 코드베이스로 작업할 때:

- 새로운 기능에 대해 카드 기반 아키텍처 유지
